# دليل عرض المستخدمين حسب حالة التسجيل

## نظرة عامة
تم إضافة نظام تبويبات جديد لعرض المستخدمين حسب حالة تسجيلهم، مع فصل قوائم الأساتذة والطلبة في كل تبويب.

## التبويبات الثلاثة

### 1️⃣ الطلبات المعلقة (⏳)
- **الغرض**: عرض طلبات التسجيل التي تنتظر الموافقة
- **المحتوى**:
  - قائمة الأساتذة المعلقين (مع أزرار الموافقة/الرفض)
  - قائمة الطلبة المعلقين (مع أزرار الموافقة/الرفض)
- **الأزرار المتاحة**:
  - ✅ الموافقة على الطلب
  - ❌ رفض الطلب

### 2️⃣ المعتمدون (✅)
- **الغرض**: عرض المستخدمين الذين تمت الموافقة على تسجيلهم
- **المحتوى**:
  - قائمة الأساتذة المعتمدين (للقراءة فقط)
  - قائمة الطلبة المعتمدين (للقراءة فقط)
- **معلومات إضافية**:
  - تاريخ الموافقة
  - اسم المسؤول الذي وافق على الطلب

### 3️⃣ المرفوضون (❌)
- **الغرض**: عرض طلبات التسجيل المرفوضة
- **المحتوى**:
  - قائمة الأساتذة المرفوضين (للقراءة فقط)
  - قائمة الطلبة المرفوضين (للقراءة فقط)
- **معلومات إضافية**:
  - تاريخ الرفض
  - اسم المسؤول الذي رفض الطلب
  - سبب الرفض

## كيفية الاستخدام

### التبديل بين التبويبات
1. انتقل إلى صفحة "👥 طلبات التسجيل"
2. في أعلى الصفحة، ستجد ثلاثة بطاقات إحصائية:
   - **⏳ طلبات معلقة**: انقر عليها لعرض الطلبات المعلقة
   - **✅ موافق عليها**: انقر عليها لعرض المستخدمين المعتمدين
   - **❌ مرفوضة**: انقر عليها لعرض الطلبات المرفوضة
3. أسفل البطاقات، ستجد أزرار التبويبات:
   - **⏳ الطلبات المعلقة**
   - **✅ المعتمدون**
   - **❌ المرفوضون**

### عرض تفاصيل المستخدمين
كل بطاقة مستخدم تحتوي على:

#### للطلبة:
- 🎓 الاسم الكامل
- 📅 تاريخ التسجيل
- تاريخ الميلاد
- مكان الميلاد
- ولاية الميلاد
- الشعبة (دراسات لغوية/أدبية/نقدية)
- التخصص (لسانيات الخطاب/تعليمية اللغات/أدب/نقد)

#### للأساتذة:
- 👨‍🏫 الاسم الكامل
- 📅 تاريخ التسجيل
- الرتبة العلمية (أستاذ محاضر/أستاذ/أستاذ مساعد)
- تاريخ الميلاد
- التخصص

## الوظائف المتاحة

### في تبويب "الطلبات المعلقة"
- **الموافقة على الطلب**: تحويل حالة المستخدم من "معلق" إلى "معتمد"
- **رفض الطلب**: تحويل حالة المستخدم من "معلق" إلى "مرفوض" مع إمكانية كتابة سبب الرفض

### في تبويب "المعتمدون"
- **عرض للقراءة فقط**: لا يمكن تعديل بيانات المستخدمين المعتمدين
- **معلومات الموافقة**: عرض تاريخ الموافقة واسم المسؤول

### في تبويب "المرفوضون"
- **عرض للقراءة فقط**: لا يمكن تعديل بيانات المستخدمين المرفوضين
- **معلومات الرفض**: عرض تاريخ الرفض، اسم المسؤول، وسبب الرفض

## الإحصائيات
في أعلى الصفحة، يتم عرض إحصائيات مباشرة:
- **عدد الطلبات المعلقة**: العدد الإجمالي للطلبات التي تنتظر المعالجة
- **عدد المعتمدين**: العدد الإجمالي للمستخدمين المعتمدين (أساتذة + طلبة)
- **عدد المرفوضين**: العدد الإجمالي للطلبات المرفوضة

## التحديثات التقنية

### التعديلات في `app.js`
1. **دالة `renderRegistrationRequests()`**:
   - تقسيم المستخدمين حسب الحالة (pending/approved/rejected)
   - تقسيم كل حالة إلى أساتذة وطلبة
   - استدعاء `renderRegistrationByStatus()` لكل حالة

2. **دالة `renderRegistrationByStatus(status, professors, students, showActions)`**:
   - عرض قوائم الأساتذة والطلبة لحالة معينة
   - التحكم في إظهار/إخفاء أزرار الإجراءات
   - عرض رسائل فارغة مناسبة عند عدم وجود بيانات

3. **دالة `renderUserCard(user, showActions, status)`**:
   - تحديث لدعم المعاملات الجديدة
   - إضافة معلومات الموافقة/الرفض
   - إخفاء الأزرار للمستخدمين المعتمدين/المرفوضين

4. **دالة `showRegistrationTab(tabName)`**:
   - التبديل بين التبويبات الثلاثة
   - إدارة الفئات النشطة للأزرار
   - إظهار/إخفاء المحتوى المناسب

### التعديلات في `index.html`
1. إضافة `onclick` للبطاقات الإحصائية:
   ```html
   onclick="showRegistrationTab('pending')"
   onclick="showRegistrationTab('approved')"
   onclick="showRegistrationTab('rejected')"
   ```

2. إضافة أزرار التبويبات:
   ```html
   <button id="tab-pending" class="tab-button active">⏳ الطلبات المعلقة</button>
   <button id="tab-approved" class="tab-button">✅ المعتمدون</button>
   <button id="tab-rejected" class="tab-button">❌ المرفوضون</button>
   ```

3. إضافة حاويات المحتوى:
   ```html
   <div id="registration-pending-container" class="reg-tab-content"></div>
   <div id="registration-approved-container" class="reg-tab-content"></div>
   <div id="registration-rejected-container" class="reg-tab-content"></div>
   ```

### التعديلات في `style.css`
1. **أنماط أزرار التبويبات**:
   ```css
   .tab-button {
       padding: 12px 24px;
       background: white;
       border: none;
       border-bottom: 3px solid transparent;
       cursor: pointer;
       font-size: 1rem;
       font-weight: 500;
       transition: all 0.3s ease;
   }
   
   .tab-button.active {
       border-bottom-color: var(--primary-color);
       color: var(--primary-color);
   }
   ```

2. **أنماط محتوى التبويبات**:
   ```css
   .reg-tab-content {
       animation: fadeIn 0.3s ease;
   }
   
   @keyframes fadeIn {
       from { opacity: 0; transform: translateY(10px); }
       to { opacity: 1; transform: translateY(0); }
   }
   ```

## ملاحظات هامة

### 🔄 المزامنة مع GitHub
- جميع التغييرات يتم حفظها في `localStorage` أولاً
- استخدم زر "🔄 مزامنة المستخدمين من GitHub" لتحميل التحديثات من الملف المشترك
- استخدم زر "📥 تصدير المستخدمين" لحفظ البيانات في ملف JSON قبل رفعه على GitHub

### 📊 تحديث البيانات
- الإحصائيات يتم تحديثها تلقائياً عند:
  - تحميل الصفحة
  - الموافقة على طلب
  - رفض طلب
  - المزامنة من GitHub

### 🔐 الصلاحيات
- فقط المسؤولون (admin) ورؤساء الأقسام (department-head, deputy-head) يمكنهم:
  - الموافقة على الطلبات
  - رفض الطلبات
  - الوصول إلى صفحة طلبات التسجيل

## استكشاف الأخطاء

### المشكلة: لا تظهر القوائم بعد التبديل بين التبويبات
**الحل**: تأكد من أن JavaScript يعمل بشكل صحيح. افتح Console في المتصفح (F12) وتحقق من عدم وجود أخطاء.

### المشكلة: البيانات لا تتطابق مع ما في GitHub
**الحل**: استخدم زر "🔄 مزامنة المستخدمين من GitHub" لتحميل آخر التحديثات.

### المشكلة: الأزرار تظهر في تبويب المعتمدين/المرفوضين
**الحل**: تأكد من أن `showActions` يتم تمريره بقيمة `false` في `renderRegistrationByStatus()` للحالات غير المعلقة.
