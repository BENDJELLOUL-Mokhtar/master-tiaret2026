# دليل مزامنة المستخدمين عبر GitHub

## المشكلة السابقة
كانت طلبات التسجيل والمستخدمين مخزنة في `localStorage` بالمتصفح فقط، مما يعني:
- ❌ لا تظهر على GitHub
- ❌ لا تظهر عند فتح المشروع من جهاز آخر
- ❌ تُفقد عند مسح ذاكرة المتصفح

## الحل الجديد
تم إضافة نظام لمزامنة المستخدمين مع ملف `users-data.json` على GitHub.

## كيفية الاستخدام

### 1. للمستخدم العادي (طالب/أستاذ)
عند فتح المشروع لأول مرة:
- ✅ يتم تحميل المستخدمين من GitHub تلقائياً
- ✅ الحقول الإضافية (تاريخ الميلاد، التخصص، إلخ) **تظهر عند اختيار الصفة**:
  - اختر **أستاذ** → تظهر: الرتبة العلمية، تاريخ الميلاد، التخصص
  - اختر **طالب** → تظهر: رقم التسجيل، تاريخ الميلاد، مكان الميلاد، ولاية الميلاد، الشعبة، التخصص

### 2. لنائب رئيس القسم
في صفحة **👥 طلبات التسجيل المعلقة**، تجد زرين جديدين:

#### أ) زر "🔄 مزامنة المستخدمين من GitHub"
- يحمّل آخر نسخة من المستخدمين من ملف `users-data.json` على GitHub
- يدمج البيانات الجديدة مع البيانات المحلية
- استخدمه عندما تريد مزامنة طلبات التسجيل الجديدة من مستخدمين آخرين

#### ب) زر "📥 تصدير المستخدمين لرفعهم على GitHub"
- يصدّر جميع المستخدمين الحاليين كملف `users-data.json`
- استخدمه بعد الموافقة على طلبات جديدة
- **خطوات رفع الملف على GitHub:**
  1. اضغط على الزر → سيُحمّل ملف `users-data.json` على جهازك
  2. افتح VS Code
  3. استبدل الملف القديم بالجديد
  4. ارفعه على GitHub:
     ```
     git add users-data.json
     git commit -m "تحديث بيانات المستخدمين"
     git push
     ```

## سير العمل الموصى به

### للإدارة (نائب رئيس القسم):
1. افتح صفحة طلبات التسجيل
2. اضغط "🔄 مزامنة" للحصول على آخر الطلبات
3. راجع وافق/ارفض الطلبات
4. اضغط "📥 تصدير" لحفظ التغييرات
5. ارفع الملف الجديد على GitHub (VS Code → git push)

### للمستخدمين الآخرين:
- فقط افتح المشروع → التحميل تلقائي ✅

## ملاحظات مهمة

### ✅ الحقول الإضافية تعمل بشكل صحيح
عند التسجيل، يجب:
1. إدخال الاسم الكامل (لقب واسم بينهما فراغ)
2. اختيار الصفة (أستاذ/طالب)
3. **ستظهر الحقول الإضافية تلقائياً** بعد اختيار الصفة
4. ملء جميع الحقول المطلوبة (*)

### 🔄 المزامنة التلقائية
- عند فتح المشروع، يتم تحميل المستخدمين من `users-data.json` تلقائياً
- إذا لم يوجد الملف أو كان فارغاً، يتم استخدام المستخدمين الافتراضيين

### 📁 الملفات الجديدة
- `users-data.json` - يحتوي على جميع المستخدمين المعتمدين
- يجب رفعه على GitHub بعد كل تحديث للمستخدمين

## استكشاف الأخطاء

**لا تظهر طلبات التسجيل الجديدة؟**
- تأكد من رفع ملف `users-data.json` الجديد على GitHub
- اضغط زر "🔄 مزامنة" في صفحة طلبات التسجيل
- امسح cache المتصفح (`Ctrl+Shift+R` أو `Cmd+Shift+R`)

**الحقول الإضافية لا تظهر عند التسجيل؟**
- ✅ هذا طبيعي! يجب **اختيار الصفة أولاً** (أستاذ أو طالب)
- بعد اختيار الصفة، ستظهر جميع الحقول المطلوبة تلقائياً

**التعديلات لا تظهر على GitHub؟**
- تأكد من حفظ الملفات في VS Code
- استخدم:
  ```
  git add .
  git commit -m "وصف التعديلات"
  git push
  ```

## الفرق بين المستخدمين والمذكرات

- **المستخدمين**: يجب رفع `users-data.json` يدوياً على GitHub
- **المذكرات**: يتم مزامنتها تلقائياً عبر `shared-data.json`
