# دليل نظام المستخدمين والصلاحيات

## نظرة عامة

تم إضافة نظام كامل لإدارة المستخدمين والصلاحيات إلى منصة إدارة لجان المناقشة. يتيح هذا النظام التحكم في الوصول إلى المنصة وتحديد ما يمكن لكل مستخدم رؤيته وفعله.

## أدوار المستخدمين

### 1. مدير المنصة (Admin)
- **المستخدم**: أ.د بن جلول مختار
- **اسم المستخدم**: `admin`
- **كلمة المرور**: `admin123`
- **الصلاحيات**:
  - عرض جميع المذكرات
  - إضافة وتعديل وحذف المذكرات
  - استيراد وتصدير البيانات
  - الوصول إلى الإحصائيات الكاملة
  - إدارة هيكل المنصة (مستقبلاً)

### 2. رئيس القسم (Department Head)
- **المستخدم**: أ.د احميدة مداني
- **اسم المستخدم**: `head`
- **كلمة المرور**: `head123`
- **الصلاحيات**:
  - عرض جميع المذكرات
  - إضافة وتعديل وحذف المذكرات
  - استيراد وتصدير البيانات
  - الوصول إلى الإحصائيات

### 3. نائب رئيس القسم (Vice Department Head)
- **المستخدم**: أ.د دبيح محمد
- **اسم المستخدم**: `deputy`
- **كلمة المرور**: `deputy123`
- **الصلاحيات**:
  - عرض جميع المذكرات
  - إضافة وتعديل وحذف المذكرات
  - استيراد وتصدير البيانات
  - الوصول إلى الإحصائيات

### 4. الأستاذ (Professor)
- **المستخدم**: د. محمد الطاهر بن عاشور
- **اسم المستخدم**: `professor`
- **كلمة المرور**: `prof123`
- **الصلاحيات**:
  - عرض المذكرات التي يشارك فيها فقط (كمشرف، رئيس لجنة، أو مناقش)
  - تصدير بياناته
  - طباعة محاضر مناقشاته
- **القيود**:
  - لا يمكنه إضافة أو تعديل أو حذف المذكرات
  - لا يمكنه رؤية مذكرات الأساتذة الآخرين
  - لا يمكنه استيراد البيانات

### 5. الطالب (Student)
- **المستخدم**: أحمد بن محمد العربي
- **اسم المستخدم**: `student`
- **كلمة المرور**: `student123`
- **الصلاحيات**:
  - عرض مذكرته الخاصة فقط
  - طباعة محضر مناقشته
- **القيود**:
  - لا يمكنه إضافة أو تعديل أو حذف المذكرات
  - لا يمكنه رؤية مذكرات طلاب آخرين
  - لا يمكنه استيراد أو تصدير البيانات
  - لا يمكنه الوصول إلى الإحصائيات

## كيفية الاستخدام

### تسجيل الدخول
1. عند فتح المنصة، ستظهر صفحة تسجيل الدخول
2. أدخل اسم المستخدم وكلمة المرور
3. اضغط على "تسجيل الدخول"
4. يمكنك الاطلاع على الحسابات التجريبية من خلال النقر على "🔑 حسابات تجريبية"

### تسجيل الخروج
- اضغط على زر "🚪 خروج" في أعلى الصفحة

### الاستمرار في الجلسة
- المنصة تحتفظ بتسجيل الدخول تلقائياً
- عند إعادة فتح المنصة، سيتم تسجيل الدخول تلقائياً
- لتغيير المستخدم، قم بتسجيل الخروج أولاً

### إنشاء حساب جديد (للأساتذة والطلبة)
يمكن للأساتذة والطلبة إنشاء حسابات جديدة:

1. **الانتقال لصفحة التسجيل:**
   - من صفحة تسجيل الدخول، اضغط على "📝 إنشاء حساب جديد"

2. **ملء البيانات المطلوبة:**
   - **الاسم الكامل**: يجب كتابة اللقب والاسم بينهما فراغ واحد
     - ✅ مثال صحيح: `بن محمد أحمد`
     - ✅ مثال صحيح: `العربي فاطمة الزهراء`
     - ❌ خطأ: `أحمد` (اسم واحد فقط)
     - ❌ خطأ: `بنمحمد` (بدون فراغ)
   - **الصفة**: اختر أستاذ أو طالب
   - **الرتبة العلمية** (للأساتذة فقط): اختر رتبتك العلمية
   - **رقم التسجيل** (للطلبة - اختياري): يمكن إدخال رقم التسجيل
   - **كلمة المرور**: يجب أن تكون 6 أحرف على الأقل
   - **تأكيد كلمة المرور**: أعد إدخال نفس كلمة المرور

3. **إنشاء الحساب:**
   - اضغط على "✅ إنشاء الحساب"
   - بعد نجاح التسجيل، سيتم توجيهك لصفحة تسجيل الدخول تلقائياً

4. **تسجيل الدخول:**
   - استخدم **الاسم الكامل** (اللقب والاسم) كاسم مستخدم
   - أدخل كلمة المرور التي اخترتها

**ملاحظات مهمة:**
- ⚠️ اسم المستخدم هو **الاسم الكامل** نفسه (اللقب والاسم)
- ⚠️ لا يمكن التسجيل باسم مستخدم مسجل مسبقاً
- ⚠️ الحسابات الإدارية (مدير المنصة، رئيس القسم، نائب) **لا يمكن إنشاؤها** من خلال التسجيل

## الواجهة حسب الدور

### واجهة المدير
- جميع عناصر القائمة مرئية
- جميع الأزرار متاحة
- يظهر اسم المستخدم ودوره في الهيدر

### واجهة الأستاذ
- قائمة محدودة (لا تظهر صفحة "إضافة مذكرة" أو "الإحصائيات")
- يرى فقط المذكرات التي شارك فيها
- أزرار التعديل والحذف مخفية
- يمكنه طباعة المحاضر فقط

### واجهة الطالب
- قائمة محدودة جداً
- يرى مذكرته الخاصة فقط
- جميع أزرار التحكم مخفية
- يمكنه فقط طباعة محضر مناقشته

## البيانات التجريبية

### تحميل بيانات تجريبية
يتوفر ملف `sample-data.js` يحتوي على 5 مذكرات تجريبية:
1. افتح Console في المتصفح (F12)
2. اكتب: `loadSampleData()`
3. حدّث الصفحة

### حذف جميع البيانات
لحذف جميع البيانات والبدء من جديد:
1. افتح Console في المتصفح (F12)
2. اكتب: `clearAllData()`
3. حدّث الصفحة

## الأمان

### ملاحظات مهمة
- ⚠️ النظام الحالي للاختبار والتطوير فقط
- ⚠️ كلمات المرور مخزنة بشكل نصي في LocalStorage
- ⚠️ لا يوجد تشفير للبيانات
- ⚠️ للاستخدام الفعلي، يجب تطوير نظام backend مع قاعدة بيانات حقيقية

### للإنتاج
عند نقل المنصة للاستخدام الفعلي، يجب:
1. استخدام قاعدة بيانات حقيقية (مثل MySQL, PostgreSQL)
2. استخدام backend آمن (مثل Node.js, PHP, Python)
3. تشفير كلمات المرور (bcrypt, argon2)
4. استخدام JWT أو Sessions للمصادقة
5. إضافة HTTPS للتشفير
6. إضافة حماية من هجمات CSRF و XSS

## إضافة مستخدمين جدد

حالياً، يتم تحديد المستخدمين في الكود:

```javascript
// في ملف app.js
const DEFAULT_USERS = [
    // أضف مستخدمين جدد هنا
];
```

لإضافة مستخدم جديد:
1. افتح ملف `app.js`
2. أضف كائن مستخدم جديد في مصفوفة `DEFAULT_USERS`
3. حدد الصلاحيات المناسبة

مثال:
```javascript
{
    id: 6,
    username: 'newuser',
    password: 'pass123',
    fullName: 'د. أحمد الجديد',
    role: 'professor',
    roleAr: 'أستاذ',
    title: 'أستاذ مساعد',
    permissions: ['view', 'export'],
    professorName: 'د. أحمد الجديد'
}
```

## استكشاف الأخطاء

### لا أستطيع تسجيل الدخول
- تأكد من صحة اسم المستخدم وكلمة المرور
- تأكد من أن الحروف الكبيرة والصغيرة صحيحة
- جرب حذف بيانات المتصفح وإعادة المحاولة

### لا أرى أي مذكرات
- تأكد من دورك (الأستاذ والطالب يرون مذكرات محدودة فقط)
- تأكد من وجود بيانات في المنصة
- جرب تحميل البيانات التجريبية

### الصفحة لا تحدّث بعد تسجيل الدخول
- حدّث الصفحة يدوياً (F5)
- امسح ذاكرة التخزين المؤقت (Ctrl+F5)
- تأكد من تفعيل JavaScript في المتصفح
